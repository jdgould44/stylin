# Stylin Link-in-Bio — Setup

Static page hosted on HostGator. Instagram grid is fed by `posts.json`, which a GitHub Action rebuilds every 15 minutes using a server-side Instagram token. No token in the browser.

## Files

- `index.html` — the page (deploy this to HostGator)
- `posts.json` — generated by the workflow, read by `index.html`
- `scripts/fetch-instagram.mjs` — fetches latest posts
- `scripts/refresh-token.mjs` — refreshes the 60-day IG token weekly
- `.github/workflows/fetch-instagram.yml` — cron every 15 min
- `.github/workflows/refresh-token.yml` — cron every Monday 8:00 UTC
- `package.json` — dev dep for the refresh script

## One-time setup

### 1. Create the GitHub repo
1. New **public** repo (public so `raw.githubusercontent.com` is reachable). Example: `stylin-link-in-bio`.
2. Push everything in this folder to it.

### 2. Wire up the raw URL in `index.html`
Open `index.html`, find this line:
```js
feedUrl: 'https://raw.githubusercontent.com/__GH_USER__/__GH_REPO__/main/posts.json',
```
Replace `__GH_USER__` and `__GH_REPO__` with your GitHub username and repo name.

### 3. Add repo secrets
In GitHub: **Settings → Secrets and variables → Actions → New repository secret**

- `IG_TOKEN` = your 60-day long-lived Instagram token
- `GH_PAT` = a Personal Access Token with **`repo`** scope (used by the weekly workflow to update `IG_TOKEN` automatically). Create at: https://github.com/settings/tokens (classic, fine-grained also works with "Secrets: Read & Write").

### 4. Kick off the first fetch
- Go to **Actions → Fetch Instagram posts → Run workflow**.
- It should commit a `posts.json` within ~30 seconds.
- After that it runs every 15 minutes automatically.

### 5. Deploy `index.html` to HostGator
Upload the updated `index.html` via File Manager. Done.

## How token refresh works

The `refresh-token.yml` workflow runs every Monday. It:
1. Calls Instagram's `refresh_access_token` endpoint → gets a new 60-day token
2. Encrypts it with the repo's public key
3. Updates the `IG_TOKEN` secret via the GitHub API

So the token never expires as long as the workflow keeps running (IG requires the token be at least 24 hours old before refresh — any fresh token works).

## Troubleshooting

- **Grid shows "Content temporarily unavailable":** `posts.json` didn't fetch. Check Actions tab for errors. Most common cause: expired `IG_TOKEN`. Manually run `Refresh Instagram token` workflow, then re-run `Fetch Instagram posts`.
- **Posts look stale:** GitHub Actions cron is best-effort; can lag 5–15 min past the scheduled time under load. You can manually `Run workflow` any time.
- **HostGator cache:** if the HTML update doesn't appear, hard-refresh (Ctrl+Shift+R).
